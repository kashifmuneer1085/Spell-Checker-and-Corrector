Startup Guide
1

Prerequisites

Make sure you have a C++17-compatible compiler installed on your system.

g++ --version # Need 7.0+ # Windows: install MinGW or use MSVC # macOS : xcode-select --install # Linux : sudo apt install g++ build-essential
2

Clone / Download the Project

Get the project files into a local folder.

git clone https://github.com/yourname/SpellCheckerPro.git cd SpellCheckerPro
3

Compile

Use make (Linux/macOS) or compile manually on Windows.

# Linux / macOS make # Manual (all platforms) g++ -std=c++17 -O2 -o spellchecker src/main.cpp
4

Run the Program

Choose a mode depending on your use case.

# Built-in word list (no dictionary file needed) ./spellchecker # With a custom dictionary file ./spellchecker dictionary.txt # Check a .txt file directly ./spellchecker dictionary.txt myessay.txt
5

Try the Interactive Commands

Once running, use the interactive prompt.

> check speling # check a word > suggest recieve # get suggestions > correct Ths is wrng # auto-correct sentence > scan She sed helllo # find all errors > demo # run built-in examples > info # dictionary stats > exit # quit
Data Structures Used

🌳 Binary Search Tree (BST)

All dictionary words are stored in a BST. Each node holds one word. Lookups traverse left/right subtrees based on lexicographic ordering, giving average-case O(log n) search.

Search: O(log n) avg · O(n) worst
Insert: O(log n) avg
In-order: O(n) → sorted output

🔤 Trie (Prefix Tree)

Built in parallel with the BST. Each node branches on one character (26 children). Enables lightning-fast prefix-based candidate generation for suggestions.

Insert: O(k) where k = word length
Prefix search: O(k + results)
Space: O(ALPHA × N × k)

📐 Edit Distance (DP)

Wagner-Fischer algorithm. Fills an (m+1)×(n+1) table to compute minimum edits (insert, delete, replace) between two strings. Used to rank all candidates by closeness.

Time: O(m × n)
Space: O(m × n)
Threshold: ≤ 3 edits for suggestions

🔗 Pipeline

When a word is not found in the BST, the Trie generates prefix candidates. These are merged with edit-distance filtered words from the BST in-order list, then ranked.

BST lookup → O(log n)
Trie prefix → O(k)
Score & rank → O(c·m·n)
